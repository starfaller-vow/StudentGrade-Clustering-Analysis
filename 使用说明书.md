# 交互式数据分析系统 - 使用说明书

## 一、系统概述

本系统是一个基于 Flask 的交互式数据分析系统，主要用于学生成绩数据的管理、清洗、可视化和聚类分析。

### 系统架构

| 模块 | 功能 |
|------|------|
| 用户认证 | 多用户登录、会话管理 |
| 文件管理 | CSV/Excel 文件上传与预览 |
| 数据清洗 | 自定义规则配置、异常值处理、缺失值填充 |
| 历史记录 | 操作日志存储与检索 |
| 数据可视化 | 图表展示（预留功能） |
| 聚类分析 | K-Means 聚类、学生等级划分 |

### 技术栈

- **后端**: Python 3 + Flask 2.x
- **数据处理**: pandas
- **机器学习**: scikit-learn (K-Means)
- **前端**: HTML5 + CSS + Bootstrap 5
- **数据库**: SQLite（历史记录）

## 三、使用流程

### 3.1 文件上传与预览

1. **进入首页** → 点击"选择文件"按钮
2. **选择文件** → 支持 `.csv` 和 `.xlsx` 格式
3. **点击上传** → 系统自动解析并展示数据预览

**支持的文件格式**:
- CSV 文件（UTF-8 或 GBK 编码）
- Excel 文件（.xlsx 格式）

### 3.2 数据清洗（核心功能）

#### 3.2.1 单次清洗流程

1. **上传文件后** → 点击"清洗规则配置"按钮
2. **配置清洗规则**:
   - 设置数值范围（最小值、最大值）
   - 选择功能开关（删除异常值、填充缺失值），删除异常值，只会删除不符合所设置的范围的数据
   - 勾选要清洗的列
3. **点击"保存并应用规则"** → 跳转到清洗结果页面
4. **查看清洗结果** → 下载清洗后的文件，注意！所下载的清洗文件我已经强制保存在outputs文件夹中

#### 3.2.2 多次清洗流程（高级功能）

系统支持对同一数据进行多次不同规则的清洗：

1. 完成第一次清洗后，点击"继续配置清洗规则"
2. 修改清洗规则（如调整数值范围、更换清洗列）
3. 再次应用清洗（效果叠加）
4. 重复步骤 1-3 直到达到预期效果

**多次清洗流程图**:

```
上传文件 → 配置规则 → 应用清洗 → 查看结果
              ↓                           ↑
         继续配置规则 ←───────────────────
```

### 3.3 清洗规则说明

| 规则项 | 说明 | 示例 |
|--------|------|------|
| **数值范围** | 设置清洗的数值边界 | 最小值=0, 最大值=100 |
| **删除异常值** | 删除超出数值范围的数据行 | 剔除分数>100或<0的记录 |
| **填充缺失值** | 用列均值填充空白数据 | NaN → 该列平均值 |
| **清洗列选择** | 指定哪些列参与清洗 | 只清洗"数学"、"总分"列 |

### 3.4 历史记录查询

1. **点击"查看我的历史记录"** → 进入历史记录页面
2. **输入关键词搜索** → 支持按用户名和操作类型搜索

## 四、功能详解

### 4.1 文件上传

- **自动编码识别**: 支持 UTF-8 和 GBK 编码的 CSV 文件
- **Excel 支持**: 直接读取 .xlsx 格式文件
- **数据预览**: 上传后立即展示表格预览

### 4.2 数据清洗规则配置

#### 数值范围设置

```
最小值: [________]    最大值: [________]
```

- 用于过滤异常值，超出范围的数据将被删除
- 适用于考试成绩等有明确范围的数据

#### 功能开关

- **删除异常值**: 勾选后删除超出范围的数据行
- **填充缺失值**: 勾选后用列平均值填充空值

#### 列选择

- 默认全选所有列
- 可根据需求只选择需要清洗的列

### 4.3 多次清洗机制

系统会保存每次清洗的结果，支持叠加清洗：

1. **第一次清洗**: 基于原始数据应用规则 A
2. **第二次清洗**: 基于清洗后的数据应用规则 B
3. **最终结果**: 规则 A + 规则 B 的叠加效果

### 4.4 文件下载

- **下载原文件**: 获取上传的原始数据
- **下载清洗后文件**: 获取经过所有清洗规则处理的数据

## 六、常见问题

### Q1: 上传文件时报错 "PermissionError"

**原因**: uploads 文件夹没有写入权限

**解决方案**:
1. 右键点击 uploads 文件夹 → 属性 → 安全
2. 确保当前用户有"写入"权限
3. 或关闭可能占用该文件夹的程序（如 Excel、资源管理器），尤其是这个，
    因为你很有可能在测试程序的时候，打开了uploads文件夹中的测试用例，这很容易导致报错

### Q2: 清洗规则配置页面不显示新文件的列

**原因**: session 中保存了旧文件的清洗结果

**解决方案**:
1. 返回首页重新上传文件
2. 或清除浏览器缓存后重试

### Q3: 多次清洗后数据变少

**原因**: 每次清洗都会根据规则删除不符合条件的数据行

**建议**:
- 第一次清洗时设置宽松的数值范围
- 逐步缩小范围进行精细化清洗

### Q4: 如何重置清洗状态

**方法**:
- 点击"返回上传页面"重新上传文件
- 系统会自动清除之前的清洗状态

## 八、注意事项

1. **数据安全**: 上传的文件会保存在 uploads 文件夹，请定期清理
2. **会话管理**: 登录状态在关闭浏览器后会自动失效
3. **文件大小**: 建议单文件不超过 10MB
4. **编码问题**: CSV 文件建议使用 UTF-8 编码
